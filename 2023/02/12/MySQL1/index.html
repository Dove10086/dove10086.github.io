关于

MySQL数据库的三大特点
1.数据独立性，拓展性
2.插入数据
3.删除数据

作用

插入数据 (insert)

语法

 insert  into 表名 values("字段1",“字段2”,"字段3","字段4","字段5");

注(AB是要录入的东西记得属性中间加逗号隔开)

数据更新 (update)

update + 数据表名字 set + 修改的属性 = 修改数值 where name =""

语法例子

update wzry set att = 200 where name ="李白"

数据删除 (delete)

语法

delete from table_name where +删除属性名字 = 删除数值;

例子

delete from wzry where age = 200;

数据选择 (select)

语法

show (展示)
databasaes (数据库)
show databases (含义展示所有数据库)

数据库组成部分

1.数据库( datbase )
2.数据库管理系统
3.数据库管理员( Administror )

数据冗余概念

多余无用的数据或重复的数据
比如四个字就可以表达的含义用40个字所讲述的意思

数据库语法

新建数据库

create database + 数据库名;

新建数据库表

create + table_nume;

打开数据库

use + 数据库名;

数据表

作用

存储数据的重要对象（一个具体的事物，一类具体的事物）

如何建立

1.界面化（可以直接用鼠标进行操作）
2.(重点掌握的) `create table  TAB`

数据模型（物理模型）

软件开发的流程

1.画出网页/客户端草图
2.写web网图
3.初始化数据库/新建数据库
4.后端代码实现
5.测试

mysql -u root =p含义：以超级管理员用户进入MySQL

数据库设计的基本步骤

1.需求分析(确定功能，软件有哪些功能)

2.概念结构设计(确定未来设计方向)

3.逻辑结构设计(确定规则)

4.物理结构设计(表结构表中字段之间的结构)

5.数据库实施(确定表数据库字段)

6.数据库运行和维护(运维数据库)

实体集

实体集：实体的属性所形成的集合就是实体集
